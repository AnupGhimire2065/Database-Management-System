Database Management System (DBMS)

  1. Data

    What is Data?

    In the context of computer science and databases, **data** refers to raw facts, figures, or unorganized information that has no specific meaning on its own. It can be in various forms such as numbers, text, images, audio, or video. Data needs to be processed, organized, and structured to become meaningful information.

    For example, "25", "Kathmandu", and "Apple" are individual pieces of data. When these are organized as "Age: 25, City: Kathmandu, Favorite Fruit: Apple", they become meaningful information about a person.

    • Data Types

      What are Data Types?

      **Data types** classify the kind of values that a variable or column can hold in a database. They determine the operations that can be performed on the data and the amount of storage space required. Using appropriate data types helps maintain data integrity and optimize database performance.

      Common Data Types in Databases:

      • **Numeric Data Types:**
        • **INT (Integer):** Stores whole numbers (e.g., 10, -500).
        • **FLOAT/DOUBLE:** Stores numbers with decimal points (e.g., 3.14, 99.99).
        • **DECIMAL/NUMERIC:** Stores exact numeric values, often used for monetary data to avoid precision issues (e.g., 123.45).
      • **String/Character Data Types:**
        • **VARCHAR(n):** Stores variable-length strings of characters, up to 'n' characters (e.g., "Hello World", "Nepal").
        • **CHAR(n):** Stores fixed-length strings of characters, padding with spaces if the string is shorter than 'n' (e.g., "A", "XYZ").
        • **TEXT:** Stores longer strings of text.
      • **Date and Time Data Types:**
        • **DATE:** Stores dates (YYYY-MM-DD) (e.g., '2025-06-22').
        • **TIME:** Stores times (HH:MM:SS) (e.g., '14:30:00').
        • **DATETIME/TIMESTAMP:** Stores both date and time (e.g., '2025-06-22 14:30:00').
      • **Boolean Data Type:**
        • **BOOLEAN/TINYINT(1):** Stores true/false or 1/0 values.
  2. Database

    What is a Database?

    A **database** is a structured collection of related data, organized in a way that allows for efficient storage, retrieval, and management. It's designed to store vast amounts of information and provide quick access to specific data. Databases are essential for almost all modern applications, from e-commerce websites to banking systems.

    Think of it as an organized digital filing cabinet where you can easily find, add, update, and delete information.

    • Types of Database

      Types of Databases

      Databases are categorized based on their underlying data model or structure. Here are some common types:

      • **Relational Databases (RDBMS):**

        These are the most common type. Data is organized into tables (relations) with rows and columns. Each table has a unique key, and relationships between tables are established using foreign keys. SQL (Structured Query Language) is used to manage and query relational databases.

        Examples: MySQL, PostgreSQL, Oracle, SQL Server.

      • **NoSQL Databases:**

        NoSQL (Not Only SQL) databases provide alternative mechanisms for storing and retrieving data that are not tabular. They are often used for handling large volumes of unstructured or semi-structured data, and for applications requiring high scalability and flexibility.

        Examples: MongoDB (Document-oriented), Cassandra (Column-oriented), Redis (Key-Value), Neo4j (Graph).

      • **Object-Oriented Databases:**

        Data is stored as objects, similar to objects in object-oriented programming languages. They are less common than RDBMS or NoSQL databases.

      • **Hierarchical Databases:**

        Data is organized in a tree-like structure, with parent-child relationships. An older type of database, less flexible than relational databases.

      • **Network Databases:**

        Similar to hierarchical but allow a more complex network structure where a child can have multiple parents. Also an older type.

  3. Software/System (Specially domain specific software)

    Domain-Specific Software/Systems

    **Domain-specific software** refers to applications designed to meet the specific needs and processes of a particular industry or business area (a "domain"). Unlike general-purpose software (like a word processor), these systems are highly tailored to perform tasks relevant to their specific field, often integrating database functionalities.

    • Database Management System

      What is a Database Management System (DBMS)?

      A **Database Management System (DBMS)** is a software system that allows users to define, create, maintain, and control access to a database. It acts as an interface between the user/applications and the database itself. A DBMS simplifies data management by providing tools for:

      • **Data Definition:** Defining the structure of the database (tables, columns, data types).
      • **Data Manipulation:** Adding, updating, deleting, and retrieving data.
      • **Data Control:** Managing user access, security, and concurrency.
      • **Data Integrity:** Ensuring the accuracy and consistency of data.

      Without a DBMS, managing large datasets would be incredibly complex and error-prone.

      Examples: MySQL, PostgreSQL, Microsoft SQL Server, Oracle Database, MongoDB.

    • Accounting System

      What is an Accounting System?

      An **accounting system** is a domain-specific software designed to record, process, and report financial transactions of a business. It automates tasks like managing ledgers, generating invoices, tracking expenses, processing payroll, and producing financial statements (e.g., balance sheets, income statements). These systems heavily rely on databases to store all financial data accurately and securely.

      Examples: Tally, QuickBooks, SAP ERP (for accounting modules).

    • Attendance System

      What is an Attendance System?

      An **attendance system** is a domain-specific software used to track and manage the attendance of employees or students. It can record time-in and time-out, calculate working hours, manage leaves, and generate attendance reports. These systems often integrate with biometric devices (fingerprint, facial recognition) or card readers to automate data collection, and they use databases to store all attendance records.

      Examples: Biometric attendance software, School/College management systems with attendance modules.

  4. Tools

    Database Tools

    **Database tools** are software applications or components that assist developers, administrators, and users in interacting with, managing, and developing databases more efficiently. These tools range from servers that host databases to languages and utilities for querying and maintaining data.

    • Server (XAMP Server)

      What is a Server (XAMPP Server)?

      In the context of web development and databases, a **server** is a computer program or device that provides functionality for other programs or devices, called "clients." For databases, a database server hosts the database and allows clients (e.g., web applications, desktop programs) to connect and interact with it.

      **XAMPP** is a popular, free, and open-source cross-platform web server solution stack. It stands for:

      • **X** (Cross-platform): Works on Windows, macOS, Linux.
      • **A**pache: The web server software.
      • **M**ySQL/MariaDB: The relational database management system.
      • **P**HP: A popular server-side scripting language.
      • **P**erl: Another scripting language.

      XAMPP provides a convenient local environment for developers to test websites and database applications without needing a live internet server. It includes Apache (for serving web pages) and MySQL (for database functionalities).

    • SQL (Structured Query Language)

      What is SQL (Structured Query Language)?

      **SQL (Structured Query Language)** is the standard language for managing and manipulating relational databases. It's used to communicate with a database to perform tasks like creating tables, inserting data, retrieving specific information, updating records, and deleting data. SQL is declarative, meaning you describe *what* you want to achieve, and the database figures out *how* to do it.

      • DDL (Data Definition Language)

        DDL (Data Definition Language)

        **DDL** commands are used to define, modify, and delete the structure of database objects (like tables, indexes, views). They deal with the schema of the database.

        Common DDL Commands:

        • **CREATE:** Used to create database objects (e.g., `CREATE TABLE`, `CREATE DATABASE`).
        • **ALTER:** Used to modify the structure of existing database objects (e.g., `ALTER TABLE ADD COLUMN`).
        • **DROP:** Used to delete existing database objects (e.g., `DROP TABLE`, `DROP DATABASE`).
        • **TRUNCATE:** Used to remove all records from a table, but keeps the table structure (faster than DELETE for clearing all rows).
        • **RENAME:** Used to rename an object.
      • DML (Data Manipulation Language)

        DML (Data Manipulation Language)

        **DML** commands are used for managing data within schema objects. They deal with the data itself.

        Common DML Commands:

        • **INSERT:** Used to add new rows (records) into a table (e.g., `INSERT INTO Customers (Name, Age) VALUES ('John Doe', 30)`).
        • **UPDATE:** Used to modify existing data in a table (e.g., `UPDATE Customers SET Age = 31 WHERE Name = 'John Doe'`).
        • **DELETE:** Used to remove rows from a table (e.g., `DELETE FROM Customers WHERE Age < 25`).
      • DQL (Data Query Language)

        DQL (Data Query Language)

        **DQL** commands are used for retrieving data from the database. Technically, `SELECT` is the primary DQL command.

        Common DQL Commands:

        • **SELECT:** Used to retrieve data from one or more tables (e.g., `SELECT * FROM Customers`, `SELECT Name, Age FROM Students WHERE City = 'Kathmandu'`).
      • DCL (Data Control Language)

        DCL (Data Control Language)

        **DCL** commands are used to control access to the database by managing user permissions and rights.

        Common DCL Commands:

        • **GRANT:** Used to give user access privileges to a database or specific objects (e.g., `GRANT SELECT, INSERT ON Employees TO user1`).
        • **REVOKE:** Used to remove user access privileges (e.g., `REVOKE DELETE ON Products FROM user2`).
      • SQL Joins

        SQL Joins

        **SQL Joins** are used to combine rows from two or more tables based on a related column between them. Joins are crucial for retrieving meaningful data from relational databases where information is often spread across multiple linked tables.

        Common Types of Joins:

        • **INNER JOIN:** Returns rows when there is a match in both tables. This is the most common type of join.
        • **LEFT (OUTER) JOIN:** Returns all rows from the left table, and the matching rows from the right table. If there is no match, NULL values are returned for columns from the right table.
        • **RIGHT (OUTER) JOIN:** Returns all rows from the right table, and the matching rows from the left table. If there is no match, NULL values are returned for columns from the left table.
        • **FULL (OUTER) JOIN:** Returns all rows when there is a match in one of the tables. If there is no match, NULL values are returned for columns from the table that doesn't have a match.

        Example: Joining an `Orders` table with a `Customers` table on `CustomerID` to get order details along with customer names.

    • CRUD Operation

      CRUD Operations (Create, Read, Update, Delete)

      **CRUD** is an acronym that stands for **Create, Read, Update, and Delete**. These are the four basic functions that are considered necessary to persist and manage data in a database. Almost every application that interacts with a database performs some combination of these operations.

      • **C - Create:** Corresponds to the **INSERT** SQL command. It's about adding new data records to the database.
      • **R - Read:** Corresponds to the **SELECT** SQL command. It's about retrieving data from the database.
      • **U - Update:** Corresponds to the **UPDATE** SQL command. It's about modifying existing data records in the database.
      • **D - Delete:** Corresponds to the **DELETE** SQL command. It's about removing data records from the database.

      Understanding CRUD operations is fundamental to database interaction and application development.

    • Keys in Database

      Keys in Database

      **Keys** are fundamental attributes or sets of attributes that are used to uniquely identify records (tuples) in a table and to establish relationships between tables in a relational database. They play a crucial role in maintaining data integrity and enabling efficient data retrieval.

      Important Types of Keys:

      • **Primary Key:**

        A column or set of columns that uniquely identifies each row in a table. A table can have only one primary key. It must contain unique values and cannot contain NULL values.

        Example: `StudentID` in a `Students` table, `ProductID` in a `Products` table.

      • **Foreign Key:**

        A column or set of columns in one table that refers to the primary key of another table. Foreign keys establish a link or relationship between two tables, ensuring referential integrity.

        Example: `CustomerID` in an `Orders` table referring to the `CustomerID` (primary key) in a `Customers` table.

      • **Candidate Key:**

        A superkey for which no proper subset is a superkey. It's a minimal set of attributes that can uniquely identify a row. A table can have multiple candidate keys, and one of them is chosen as the primary key.

        Example: Both `StudentID` and `PassportNumber` could be candidate keys for a `Students` table if both are unique.

      • **Super Key:**

        A set of one or more attributes that, taken collectively, can uniquely identify a row in a table.

        Example: If `StudentID` is a primary key, then `(StudentID, Name)` is a super key.

      • **Alternate Key:**

        A candidate key that is not chosen as the primary key.

    • Constraints in Database

      Constraints in Database

      **Database constraints** are rules enforced on data columns of a table to limit the type of data that can be entered into it. They are used to ensure the accuracy and reliability of data in the database, maintaining data integrity.

      Common Types of Constraints:

      • **NOT NULL:** Ensures that a column cannot have a NULL value.
      • **UNIQUE:** Ensures that all values in a column are different. A table can have multiple UNIQUE constraints.
      • **PRIMARY KEY:** A combination of NOT NULL and UNIQUE. Uniquely identifies each record in a table. (Discussed above)
      • **FOREIGN KEY:** Ensures referential integrity between two tables. (Discussed above)
      • **CHECK:** Ensures that all values in a column satisfy a specific condition (e.g., `CHECK (Age >= 18)`).
      • **DEFAULT:** Sets a default value for a column when no value is specified during an INSERT operation.
    • Anomolies in Database

      Anomalies in Database

      **Database anomalies** are flaws or inconsistencies that can arise in a database when data is not properly organized or normalized. They typically occur in poorly designed tables where redundant data exists. Anomalies lead to data inconsistency and make database management difficult.

      Common Types of Anomalies:

      • **Insertion Anomaly:**

        Occurs when you cannot insert a new record into a table without also inserting information about something else that is not yet fully available or relevant.

        Example: In a table storing student details and their enrolled courses, you might not be able to add a new course until a student enrolls in it, or you can't add a student without assigning them to a course.

      • **Deletion Anomaly:**

        Occurs when deleting a record results in the loss of other, seemingly unrelated, information that is still needed.

        Example: If you delete the last student enrolled in a particular course from a combined student-course table, you might unintentionally delete all information about that course.

      • **Update Anomaly:**

        Occurs when a change to one data item requires multiple updates to maintain consistency across the database, and if all copies are not updated, data inconsistency arises.

        Example: If a professor's department name is stored multiple times for each course they teach, changing their department requires updating multiple records. Failure to update all instances leads to inconsistent data.

      These anomalies are primarily addressed through database normalization.

    • Normalization in Database

      Normalization in Database

      **Normalization** is a systematic process of organizing the columns and tables of a relational database to minimize data redundancy and improve data integrity. The goal of normalization is to decompose larger, "badly" structured tables into smaller, "well-structured" tables without losing any information.

      It involves a series of guidelines called **Normal Forms (NF)**. The most commonly used normal forms are:

      • **First Normal Form (1NF):**

        Eliminates repeating groups/attributes. Each column must contain atomic (single, indivisible) values, and each row must be unique.

      • **Second Normal Form (2NF):**

        Must be in 1NF and all non-key attributes must be fully functionally dependent on the primary key. (Eliminates partial dependencies).

      • **Third Normal Form (3NF):**

        Must be in 2NF and eliminate transitive dependencies. No non-key attribute should be functionally dependent on another non-key attribute.

      • **Boyce-Codd Normal Form (BCNF):**

        A stronger version of 3NF. It addresses certain types of anomalies not covered by 3NF, particularly when a table has multiple overlapping candidate keys.

      Normalization helps to avoid anomalies, reduce storage space, and make the database more flexible and easier to maintain.

    • Database Security

      Database Security

      **Database security** refers to the comprehensive measures taken to protect the database and its contents from unauthorized access, malicious attacks, corruption, or data loss. It's crucial for maintaining the confidentiality, integrity, and availability (CIA triad) of an organization's data.

      Key Aspects of Database Security:

      • **Access Control:**

        Restricting who can access the database and what actions they can perform (e.g., read, write, update, delete). This involves user authentication (verifying identity) and authorization (granting specific permissions).

      • **Encryption:**

        Converting data into a coded format to prevent unauthorized access. Data can be encrypted at rest (when stored) and in transit (when being transmitted).

      • **Auditing and Monitoring:**

        Tracking and logging all database activities to detect suspicious behavior, identify potential breaches, and ensure compliance.

      • **Backup and Recovery:**

        Creating copies of the database to restore it in case of data loss due to hardware failure, corruption, or accidental deletion.

      • **Data Masking/Obfuscation:**

        Hiding original data with random characters or other data to protect sensitive information while still allowing for testing or development.

      • **Firewalls and Network Security:**

        Protecting the network perimeter and the database server from external threats.

      • **Regular Security Updates:**

        Applying patches and updates to the DBMS software to fix known vulnerabilities.
